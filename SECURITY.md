# Security Policy

## Reporting a vulnerability

Please report security or privacy issues privately to the repository owner before opening a public issue.

This repository is public and should never contain:

- secrets, tokens, cookies, private keys, or `.env` files;
- private knowledge-base content;
- customer data, proprietary project data, tickets, or internal URLs;
- private screenshots or personal documents.

## Public data boundary

The repository contains public-safe style references, prompt templates, generated visual assets, and provenance records for `viggomeesters.com`.

If a copied design artifact contains private context, remove it before publishing or replace it with a public-safe derivative.
